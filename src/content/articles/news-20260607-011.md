## ポイント

・AlibabaグループがAIコードレビューツール「Open Code Review」をOSS公開、社内数万人の開発者が利用し累計100万件のコード欠陥を検出
・既存AIモデルにルールセットを追加することでレビュー精度を向上させる仕組みで、特定言語・フレームワークへのカスタマイズが可能
・AIによるコード生成が普及する一方、レビュー・品質管理の自動化も急速に整備されつつある

---

## 億速の見解

「AIがコードを書ける時代に、プログラミングを勉強する意味はあるの？」という問いを会社員から聞くことが増えた。この問いに対する私の見方は「書く力より読む力と設計力が重要になった」だ。

Open Code Reviewのようなツールが示しているのは、コードの生成だけでなく「レビュー」までAI化が進んでいるという現実だ。つまり、AIが書いたコードをAIが検査する時代が来ている。この流れの中で人間に残る仕事は「何を作るかの判断」と「システム全体の設計・責任の所在の確認」になっていく。

副業・スキルアップ目的でプログラミングを学ぶ会社員は、「コードを書き切る」より「AIアウトプットを評価・修正・改善する」スキルを重視すべきフェーズに入っている。ゼロからコーディングできなくても、仕様書を書ける・バグを論理的に説明できる・コードの意図を読み解ける人間の価値は、むしろ上がっているとさえ感じる。
